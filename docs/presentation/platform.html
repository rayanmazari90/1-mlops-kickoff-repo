Chapter 03 · Divide & conquer

Platform: config → pipeline → registry → API

We didn’t bolt on a random API. The same src/ modules power training, evaluation, batch inference, and serving — so there is one brain, many surfaces.

System architecture

Memorize this box for the examiners: orchestration in main.py, promotion in W&B, consumption via FastAPI and Streamlit.

┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ │ config.yaml │───>│ main.py │───>│ W&B │───>│ api.py │ │ .env │ │ orchestrator│ │ tracking │ │ FastAPI │ └─────────────┘ │ │ │ registry │ │ /health │ │ load_data │ │ artifacts │ │ /predict │ │ clean_data │ └──────┬──────┘ └──────┬──────┘ │ validate │ │ prod │ │ features │ │ alias │ │ train │ ▼ │ │ evaluate │ ┌─────────────┐ ┌──────▼──────┐ │ infer │ │ model │ │ Render │ └─────────────┘ │ joblib │ │ Docker │ └─────────────┘ └──────┬──────┘ ┌─────────────┐ │ │ app.py │──────────────────────────────┘ │ Streamlit │ calls /predict └─────────────┘

Code map (what each file “sells”)

main.py — Single entry; W&B init; full orchestration.
load_data.py — Ingest ATP CSVs; idempotent pulls.
clean_data.py — Standardise names, dates, ranks.
validate.py — Pandera: fail fast on bad data.
features.py — Leakage-aware features + sklearn recipe.
train.py / evaluate.py — Fit, metrics, baseline compare.
infer.py — Batch scoring with strict schema.
api.py — Zero ML logic; Pydantic I/O only.
app.py — Streamlit; real player presets; UX.
logger.py — Dual log; no print() in prod.

API contract (soundbite)

POST /predict  →  { "prediction": 1, "probability": 0.73 }
GET  /health   →  { "status": "ok", "model_loaded": true }

OpenAPI is auto-generated — Swagger and ReDoc are your free “sales brochure” for technical buyers.
