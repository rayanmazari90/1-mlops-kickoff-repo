Chapter 01 · The scoreboard

If you can’t measure it, you can’t trade it

In pre-match analytics, raw accuracy is a vanity metric. Desks care about calibrated probabilities (log loss, Brier) because those drive position sizing and risk. Below: business KPIs we optimise for operationally, then model KPIs from our held-out test set vs a rank-only baseline.

Business KPI framework (how we sell value)

These are the levers a sports analytics or trading-adjacent stakeholder actually feels — aligned with our README business case.

Business KPI What “good” looks like How we deliver
Calibration quality Probabilities that match reality (not just direction) Optimise log loss & Brier vs baseline; report both in evaluation
Decision latency Seconds from question → answer, not “wait for the analyst’s notebook” REST /predict + Streamlit for non-technical users
Model governance One promoted production model; traceable lineage W&B registry + prod alias; API loads that artifact on startup
Change confidence New code doesn’t silently break predictions CI: black, flake8, pytest with ≥90% coverage on core src/
Operational audit trail When something fails, we know where to look Structured logging; W&B runs; Render API logs
